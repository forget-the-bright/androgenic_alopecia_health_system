## 1. 项目概述
### 1.1 项目背景
本系统针对男性雄激素性脱发患者，提供**毛发数据记录、AI智能分析、用药管理、用户权限管控**一体化服务，实现患者脱发情况可视化追踪、规范化管理，同时支持管理员进行系统运维与数据监管。

### 1.2 核心价值
1. 患者端：自动归档毛发照片、AI辅助评估毛发变化、标准化用药管理；
2. 管理端：统一用户管理、全局数据统计、系统资源配置；
3. 技术端：安全隔离用户数据、可靠的对象存储、标准化接口与权限管控。

### 1.3 技术栈总览
| 层级 | 技术选型 |
| ---- | ---- |
| 后端框架 | Spring Boot |
| 权限安全 | Sa-Token |
| ORM框架 | MyBatis-Plus |
| 前端架构 | 前后端一体、Thymeleaf、Bootstrap、jQuery |
| 接口文档 | Swagger/Knife4j |
| 文件存储 | MinIO分布式对象存储 |
| 数据库 | MySQL（企业级关系型数据库） |

## 2. 功能需求
### 2.1 通用功能
#### 2.1.1 多语言切换
- 支持中文、英文等多语言一键切换；
- 页面文本、提示信息、按钮文字同步适配语言配置；
- 语言选择持久化，下次登录自动生效。

#### 2.1.2 账户体系
- 支持用户注册、登录、退出、密码修改；
- 基于Sa-Token实现会话管理、自动登录、会话超时控制；
- 双角色体系：**普通用户**、**系统管理员**。

### 2.2 普通用户功能
#### 2.2.1 头发数据管理
1. **照片上传**
   - 支持头部多部位（头顶、前额、两侧等）照片上传；
   - 自动记录上传日期，按**天+部位**自动分组；
   - 图片格式限制：JPG、PNG，大小限制：≤10MB；
   - 上传后自动同步至MinIO指定Bucket，返回文件访问地址。
2. **数据展示**
   - 按时间轴展示每日毛发照片数据；
   - 按部位筛选查看对应历史数据；
   - 支持照片预览、删除、重命名。
3. **数据管理**
   - 单条/批量删除照片数据；
   - 自动关联照片存储地址，删除数据同步清理MinIO文件。

#### 2.2.2 AI对比分析
- 支持选择**两张历史照片**发起AI对比分析；
- 系统调用外部AI接口，实现毛发密度、生长情况智能识别与评估；
- 展示AI分析报告：评分、变化趋势、专业解读；
- 历史分析报告永久保存，支持重复查看。

#### 2.2.3 用药管理
1. **用药方案自定义**
   - 添加用药名称、剂量、服用时间、用药周期；
   - 支持多种药物同时管理；
   - 编辑、停用、删除用药方案。
2. **每日用药打卡**
   - 日历式打卡界面，直观展示用药记录；
   - 支持补打卡、标记漏服；
   - 自动统计用药完成率、连续打卡天数。

#### 2.2.4 个人资料
- 查看/修改个人基础信息；
- 账号安全设置（密码修改）；
- 语言偏好设置。

### 2.3 管理员功能
#### 2.3.1 用户管理
- 查看所有用户列表（分页、搜索、筛选）；
- 查看用户详情（基础信息、毛发数据、用药数据统计）；
- 禁用/启用用户账号、重置用户密码；
- 删除非法用户及关联数据。

#### 2.3.2 数据统计与管理
- 全局用户数量、照片上传总量、AI分析次数统计；
- 查看所有用户毛发照片与分析报告；
- 数据异常监控、非法数据清理。

#### 2.3.3 系统配置
- MinIO存储配置查看（Bucket信息、存储容量）；
- 外部AI接口配置（地址、密钥、调用次数）；
- 系统日志查看、系统参数配置。

### 2.4 数据安全需求
- 严格的数据隔离：普通用户仅可访问自身所有数据；
- 照片数据加密存储，仅授权用户可访问；
- 操作日志记录，关键操作留痕；
- 敏感接口权限校验，防止越权操作。

## 4. 数据存储设计
### 4.1 MinIO对象存储
- 独立Bucket：`hair-loss-images` 存储所有用户毛发照片；
- 文件命名规则：`用户ID/部位/日期-时间戳.后缀名`；
- 支持文件预览、删除、权限控制；
- 提供临时访问URL，保障数据安全。

### 4.2 数据库核心表结构
1. **用户表（sys_user）**：用户ID、账号、密码、姓名、角色、语言、创建时间等；
2. **毛发照片表（user_hair_image）**：ID、用户ID、部位、文件URL、MinIO路径、上传日期、备注等；
3. **AI分析记录表（ai_analysis）**：ID、用户ID、对比照片ID、AI报告、评分、分析时间等；
4. **用药方案表（user_medicine）**：ID、用户ID、药物名称、剂量、用法、状态等；
5. **用药打卡表（medicine_clock）**：ID、用户ID、药物ID、打卡日期、打卡状态等；
6. **系统配置表（sys_config）**：配置项、配置值、备注。

### 5.1 接口文档
- 集成Knife4j，访问地址：`项目地址/doc.html`；
- 所有接口必填：接口说明、请求参数、响应参数、错误码；

## 7. 权限体系设计
基于Sa-Token实现细粒度权限控制：
1. **普通用户**：
   - 权限：自身数据增删改查、AI分析、用药管理；
   - 禁止：访问其他用户数据、管理员功能。
2. **系统管理员**：
   - 权限：所有用户管理、全局数据查看、系统配置、日志管理；
   - 禁止：修改用户隐私数据（密码除外）。
3. **会话控制**：
   - 登录超时自动退出；
   - 单账号多地登录自动下线；
   - 敏感操作二次验证。
